# v0.5.3 Hardware Testing Feedback

> Captured 2026-06-01 from user hardware testing.

Status legend: `[ ]` open · `[x]` done · priority: P0=critical / P1=important / P2=nice-to-have

---

## Android

- [x] **HW-A1 — Private mode logic inverted.** When private mode is ENABLED, clips start
  being added. When DISABLED (default), clips are blocked. Logic is inverted in the
  service/repository. Default is also still wrong on existing installs (stale SharedPrefs).
  *(fixed: dccb340)*

- [x] **[P0] HW-A2 — Clip capture stopped working in-app entirely.** App no longer saves clips
  made within the app itself (without accessibility). Regression — possibly AND8 changes
  or private mode inversion.
  *(fixed: AccessibilityService init no longer gated behind sync-init try/catch; settings/repository
  initialized unconditionally; syncManager made nullable so a sync-init failure can't kill capture.)*

- [x] **[P1] HW-A3 — Clip duplicates on tap.** Tapping a clip in the list causes it to appear
  twice. Likely `loadItems()` called twice on copy.

- [x] **[P1] HW-A4 — QR scan button missing.** The button to SCAN another device's QR code
  (camera, to initiate pairing) was removed. Needs to be restored in the Devices flow.

- [x] **HW-A5 — QR re-blurs after second tap.** Second tap regenerates the QR but sets
  `qrBlurred = true` again — should stay visible (unblurred) after regeneration.
  *(fixed: dccb340)*

- [x] **[P2] HW-A6 — Duplicate text under QR.** Two labels shown: "Expires in N s" AND
  "Token expires in 60 seconds. Keep this screen visible." Remove the second one.

- [x] **HW-A7 — QR does not auto-regenerate on expiry.** When the token expires, the
  user must manually trigger regeneration. Should auto-regenerate with a countdown timer.
  *(fixed: dccb340)*

- [ ] **[P2] HW-A8 — Search field too short.** Search should be hidden behind an icon button
  and expand to full-width when activated. Show last 5 recent searches as suggestions.

- [x] **[P1] HW-A9 — P2P settings missing.** The P2P sync toggle is gone from SettingsActivity.
  Check which tab it was moved to or if it was accidentally deleted.

- [x] **[P2] HW-A10 — Unsaved settings warning missing.** If user exits settings without
  saving, show a confirmation dialog ("Discard changes?").

- [ ] **[P2] HW-A11 — ADB command wrong/incomplete.** The ADB command shown in the
  accessibility setup screen is not complete. Should match the ClipCascade format
  (full adb shell command for granting clipboard access). Must be copyable on tap.

- [x] **[P2] HW-A12 — Google Lens QR does not open app.** Scanning the pairing QR via
  Google Lens does not associate with the app. Missing intent filter / deep link for
  the `CPPAIR1.…` URI scheme.

## macOS

- [x] **[P2] HW-M1 — Install with running daemon.** Installing a new version while the daemon
  is running causes conflicts. Need to either: kill the daemon before install, or detect
  and prompt user to quit it first.

- [x] **[P2] HW-M2 — macOS missing pairing QR flow.** macOS does not have a "show QR to be
  scanned by Android" flow that is easily accessible. (Verify if DevicesView already
  has it or if it's hidden.)

- [x] **[P2] HW-M3 — Description tooltips overflow/clip.** Info popover text gets clipped by
  its container. Popover must render above/outside the container (z-index + overflow visible).

- [x] **[P1] HW-M4 — Slider lost accent color.** Settings sliders lost their colored track.
  Regression from replacing StepSlider with SliderRow — `accent-ide-accent` Tailwind class
  has no effect when `appearance: none` is set. Fix: compute fill% and set `--_fill` CSS var
  on the input; update index.css track gradient to use it.

- [x] **[P2] HW-M5 — Popup: pinned item date/buttons misaligned.** For pinned items in the
  popup, the timestamp and action buttons are not aligned (M1 regression).

- [x] **[P0] HW-M6 — Popup copies unreliably.** Clips copy only every other click, or
  intermittently fail. Images never copy from popup. Root cause: likely race between
  onBlur-triggered hide and onClick copy flow, or image content_type write error silently
  swallowed (popup already hidden, error toast invisible). Needs hardware repro to confirm.

- [x] **[P2] HW-M7 — No notification when copying from main window.** Sound/notification only
  fires from popup copy, not from main window copy. (Related to M12 wiring — may need
  daemon-side trigger re-check.)

- [x] **[P2] HW-M8 — Not all setting descriptions moved to tooltips.** Some settings in
  SettingsView still show inline description text instead of the info popover.

- [x] **[P2] HW-M9 — About page: last repo link looks broken/unstyled.**

## Both platforms

- [x] **[P1] HW-B1 — M3 regression: pinned-item background.** Pinned item background
  distinction may have regressed. Was marked done but user reports it's missing.

- [ ] **[P0] HW-B2 — Device pairing not working.** No devices can be paired (QR issues +
  no camera scan button on Android + unclear macOS flow). This blocked most of
  the hardware testing.

- [x] **HW-A13 — Clear All still deletes pinned items (Android).** A2 fix only patched
  the macOS daemon SQL. Android `ClipboardRepository` does direct SQLite — its `deleteAll()`
  has no `WHERE pinned = 0` guard. Root cause: Android bypasses daemon for local ops.
  *(fixed: 356ea36)*

- [x] **[P1] HW-A14 — Image quality setting has a separate Save button.** Should be saved
  with the rest of settings via the main Save button (AND4). Regression or incomplete fix.

- [x] **HW-A15 — Pinned items jump to top when copied.** When copying a pinned clip,
  it moves to position 0. Pinned items must stay in their pinned position regardless
  of copy actions.
  *(fixed: 356ea36)*

- [x] **HW-A16 — Reorder (A1) not working on Android.** Button is visible but ↑/↓
  does not actually reorder items. Pinned items also move when copied (wall_time sort
  overrides pinnedSortIndex).
  *(fixed: drag-to-reorder 8d25f47 + position-on-copy 356ea36)*

- [x] **[P1] HW-A17 — AND5/AND10 issues.** User flagged AND5 (image height slider) and
  AND10 (fingerprint copy) still have problems — needs investigation.

## Notes

  capture; "disabling" blocks it. Fix must invert the boolean check in the service.
- Device pairing blocked all sync testing. Top priority for next fix wave.
- ADB accessibility command: reference the ClipCascade app format for the correct
  command structure.
