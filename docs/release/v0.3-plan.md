# CopyPaste v0.3.0 — Development Plan

_Status: draft · cycle opens 2026-05-23 · cut targets `v0.3.0-beta.1`_

This document scopes the v0.3.0 release cycle. It is derived from:

- `docs/security/THREAT-MODEL.md` §7 (Open Issues OI-1..OI-10)
- `docs/release/RELEASE-CHECKLIST.md` (items deferred during v0.2 cuts)
- `docs/privacy/telemetry-policy.md` (telemetry roadmap)
- In-source `TODO(v0.3)` markers, primarily `crates/copypaste-core/src/crypto/encrypt.rs`
- Completed-task ledger from `.claude/scripts/task-queue.sh list completed`
- Recent commit titles flagged "beta-bonus" / "deferred" / "follow-up"

---

## Goal

v0.3.0 promotes CopyPaste from a beta usable on macOS + Android (with a
text-only Slint UI and a Sentry _stub_ telemetry crate) into a **multi-platform
clipboard-sync product with production-grade security hygiene, real opt-in
telemetry, and a credible Windows experience**. The release closes the
medium-severity items left open in the v0.2 threat model, retires the legacy
empty-AAD ciphertext fallback that v0.2 had to keep around for in-place
upgrades, and ships the first Slint build for Windows so the platform matrix
stops being _"macOS done, Android done, Windows TBD"_.

It is **not** a 1.0 candidate. Peer revocation (OI-2), traffic-analysis
padding (OI-5), and Apple notarization remain post-v0.3 work — explicitly
deferred below.

---

## Themes

1. **T1 — Security backlog burn-down.** Close the MEDIUM-severity items from
   THREAT-MODEL §7 (OI-3 mDNS rate-limit, OI-4 Keychain ACL, OI-6 Sentry
   scrubbing CI gate, OI-9 `unsafe` review). Retire the legacy
   empty-AAD decrypt fallback (`encrypt.rs` `TODO(v0.3)`).
2. **T2 — Telemetry pipeline GA.** Replace the `unimplemented!()` Sentry
   stub in `copypaste-telemetry` with a real opt-in pipeline: minimal-vs-full
   consent honored, PII-scrubbing transport, CI assertion that no A1/A2/A3
   payloads ever escape, daemon/CLI/UI consent UX. _Default remains
   `Disabled`._
3. **T3 — Windows UI parity (Slint port).** Bring the Slint UI up on Windows:
   cross-compile via the cross-builder image that already ships sqlcipher
   + openssl deps (commit `ab6119f`), wire tray icon + autostart registry
   entry, and add `windows-latest` to the CI matrix. macOS / Android remain
   reference platforms; Linux is out of scope this cycle.
4. **T4 — Pairing & multi-device UX polish.** Token-expiry countdown in
   Android `PairActivity`, pluralisable strings via `ngettext`, settings
   surface for revoking individual paired peers (manual delete-entry flow,
   not full OI-2 revocation protocol).
5. **T5 — Schema v3 → v4 with HKDF v2 + ciphertext migration.** The ignored
   tests in `tests/hkdf_versioning.rs` (`HKDF_SALT_V2`,
   `derive_enc_key_v2`, `migrate_ciphertext_v1_to_v2`) become real:
   ship the v1→v2 ciphertext migration so v0.4 can drop v1 decrypt entirely.

## Security backlog — OI mapping

| OI | Severity | v0.3 disposition |
|----|----------|------------------|
| OI-1 | LOW | **Closes in v0.3** (1.3 — `SCHEMA_VERSION` made `pub` as part of T5.3) |
| OI-2 | HIGH | **Deferred to 1.0.** Manual per-peer delete-entry UX (4.3) lands in v0.3 as interim mitigation; full revocation protocol with epoch numbers and re-pairing remains 1.0 scope. |
| OI-3 | MEDIUM | **Closes in v0.3** (1.2) |
| OI-4 | MEDIUM | **Closes in v0.3** (1.3) |
| OI-5 | LOW | **Deferred post-stable.** Padding + cover traffic require protocol-level changes that don't fit v0.3. |
| OI-6 | MEDIUM | **Closes in v0.3** (1.4 + T2 in aggregate — the CI gate plus the real scrubber together) |
| OI-7 | LOW | **Docs-only.** Update `docs/privacy/threat-model-faq.md` to call out OS-level clipboard managers as a user responsibility. |
| OI-8 | LOW | **Closes in v0.3** (1.7) |
| OI-9 | MEDIUM | **Closes in v0.3** (1.5) |
| OI-10 | LOW | **Closes in v0.3** (1.6) |

Closing 6 of 10 (and explicitly downgrading 3 of the remaining to
LOW/deferred) is the success criterion for the security theme.

candidates (already listed above where applicable):

- Telemetry real backend (was `unimplemented!()` stub) — see T2
- Slint Windows port — see T3
- `task-settings-pair-windows` — see 3.5
- `task-history-window-ipc` — see 4.4
- HKDF v2 + ciphertext migration — see T5
- `pub SCHEMA_VERSION` (OI-1 cleanup) — folded into 5.3

  + signing-cert workflow that we still haven't budgeted)

  notarization)

  frozen for v0.3. We will revisit after v0.3 ships.
- **Relay v2 quotas** (`task-relay-v2-quotas` pending) — defer to v0.4.
- **Apple notarization + Sparkle** — see "Deferred from v0.2".

## Cut criteria

`v0.3.0-beta.1` is taggable when **all** of these are true:

- [ ] All T1 tasks (1.1–1.7) merged on `release/v0.3.0-beta` and CI green
- [ ] All T2 tasks (2.1–2.4) merged; `telemetry-consent` defaults to
      `Disabled` and CI gate 1.4 is mandatory
- [ ] T3.1 + T3.3 + T3.4 merged; `windows-latest` job green on every PR
- [ ] T5.1 + T5.2 + T5.3 merged; v3→v4 migration tested on a fixture DB
      seeded with v0.2 data (no data loss, no plaintext)
- [ ] OI-9 `unsafe` review document committed at `docs/security/unsafe-review-v0.3.md`
- [ ] No `TODO(v0.3)` strings remain in `crates/copypaste-core/src/crypto/`
- [ ] `THREAT-MODEL.md` §7 updated: OI-1, OI-3, OI-4, OI-6, OI-8, OI-9,
      OI-10 marked as closed with commit refs
- [ ] `RELEASE-CHECKLIST.md` walked end-to-end successfully on a dry-run
- [ ] Release notes drafted from `git-cliff` since `v0.2.0-beta.1`
- [ ] Test suite passes on macOS-latest **and** windows-latest;
      Android instrumented tests green on at least one API-30+ emulator

Soft criteria (nice-to-have, do not block tag):

- 4.3 (per-peer revocation UI) — if not ready, ship 4.3 in `beta.2`
- 3.2 (Windows tray icon) — text-only daemon-with-no-tray is acceptable for
  `beta.1`; tray can land in `beta.2`
