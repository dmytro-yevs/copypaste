# v0.3.0 — Release Plan

- **Status:** In progress (cut from `v0.2.0-beta` HEAD on 2026-05-22)
- **Branch:** `release/v0.3.0-dev`
- **Target:** Q3 2026
- **Supported platforms:** macOS (arm64 + x86_64), Android (arm64-v8a)
- **Distribution:** Homebrew Cask only (see ADR-010, ADR-012)

---

## Scope

v0.3 closes the cryptographic-substitution gap left by the v0.2 empty-AAD
decrypt fallback and lands the first round of UX/UI polish on the two
supported platforms (macOS + Android). It is **explicitly not** a port
release — Windows remains frozen (see ADR-012).

### Themes (4)

| # | Theme                  | Status      |
|---|------------------------|-------------|
| T1 | Crypto hardening      | In progress |
| T2 | Sync robustness (LAN + relay) | Planned |
| T3 | macOS UI polish (Slint) | Planned   |
| T4 | Android UX polish     | Planned     |

### Features

- **T1 — Crypto:** drop legacy empty-AAD AEAD fallback (BREAKING — done in 1c55e57)
- **T1 — Crypto:** AAD-bound encryption mandatory across all `encrypt_item`
  call sites (done across daemon, android, benches)
- **T2 — P2P:** mDNS per-IP token-bucket rate limit (10 req/2 s, done in 2dc0a25)
- **T2 — Pairing:** Android pairing-token expiry countdown + auto-cancel UI (done in 2f93e62)
- **T3 — macOS UI:** history search affordance, sensitive-item redaction toggle,
  tray menu polish
- **T4 — Android:** sensitive-item visual marker, pairing flow QA pass,
  foreground-service notification redesign

### Removed

- **`copypaste-config` crate** — DELETED (v3-config-delete, DONE). Zero
  production consumers; functionality is orthogonal to `core::config::AppConfig`
  and `daemon::ipc::AppConfig`. See ADR-011 for rationale.

### Deferred

- **Slint Windows port** — DEFERRED (Windows frozen 2026-05-23, see ADR-012).
  The platform abstraction (`crates/copypaste-daemon/src/platform/windows.rs`)
  and named-pipe IPC skeleton (`crates/copypaste-daemon/src/ipc_win.rs`) are
  retained but not built. Thaw is gated on (a) Slint Windows port becoming
  maintainable and (b) Apple Developer ID acquisition unblocking the wider
  signing/distribution question.
- **iOS** — not in scope (frozen per platform policy; no active plan).

## Cut criteria

All of the following must be true before tagging `v0.3.0`:

1. `cargo test --workspace --all-features` is green on macOS arm64 and on
   the Ubuntu container (relay + core).
2. `cargo clippy --workspace --all-targets -- -D warnings` is clean.
3. `cargo audit` has zero un-waived advisories.
4. CHANGELOG entry written (no auto-generated stub).
5. Threat model re-reviewed (no new asset added since v0.2 review).
6. Homebrew Cask smoke install on a clean macOS host (arm64 + x86_64).
7. Android `.aar` builds via `scripts/build-in-docker.sh android` and the
   demo APK launches on a clean emulator.

Removed for v0.3:

- ~~Windows CI matrix gate~~ — Windows frozen (ADR-012).
- ~~Windows host or Wine smoke install~~ — same.

## Distribution

Homebrew Cask only. See `docs/release/RELEASE-CHECKLIST.md` §5 and ADR-010.

Explicitly NOT used in v0.3:

- Apple notarization (no Developer ID per project policy).
- Sparkle / autoupdate feed.
- Standalone `.dmg` direct downloads from the GitHub release page as a
  promoted install path (the file still ships as a release asset for
  reproducibility, but the README points users at the Cask).
