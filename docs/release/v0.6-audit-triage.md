## Cluster A — LAN/SAS pairing functional bugs (the feature is broken after first use)
- **A1 (P1) initiator never resets SM.** `ipc.rs` `pair_with_discovered` calls `finish(Confirmed|Rejected)` but never `reset()` → SM stuck terminal → every subsequent pair returns `ERR_CODE_RATE_LIMITED`. Fix: `reset()` in both arms after recording outcome. (daemon)
- **A2 (P1) responder reset-race hangs the macOS modal.** Responder resets to `Idle` immediately after `finish`; UI polls `pair_get_sas` every 700ms and can observe `idle` (neither success nor failure) → spins forever on spinner. Fix (UI, sufficient): poller treats `idle` observed *after* a non-idle state as terminal. Optional better fix: defer responder `reset()`. (ui [+ daemon optional])
- **A3 (P1) SAS modal poll-effect timer leak.** `DevicesView.tsx:339` effect keyed on inline `onPaired` (`:1232`) re-created every ~3s parent re-render; shared `cancelledRef` lets stale closures keep rescheduling ~700ms polls. Fix: pass stable `onPaired={loadPeers}` + per-effect local `cancelled`. (ui)

## Cluster B — Settings single-source-of-truth (user's explicit requirement)
- **B1 (P0) Android does NOT participate in AppConfig.** UniFFI exposes no config fns; `Settings.kt` hand-mirrors every setting in SharedPreferences with its own defaults + (mostly) no clamp. Root of all parity gaps. Fix: expose `get_config()/set_config(Config)` over UniFFI backed by `AppConfig`+`clamp_values()` (ABI bump + bindings regen); Settings.kt reads canon via FFI. (android crate — LARGE)
- **B2 (P0) retention ~100× divergence.** Android prunes history at 1000 items (`maxHistoryItems`) / 200 (`historySize`); macOS is byte-only (`storage_quota_bytes`). Fix: drive Android retention from `storage_quota_bytes`; retire count caps (or display-only). (part of B1)
- **B3 (P1) file-size cap incoherence + bug.** Five literals: 1 GiB (config default) / 100 MiB (`file.rs` hard) / 8 MiB (`SYNC_MAX_BLOB_BYTES`) / 10 MiB (relay) / 16 MiB (frame). UI advertises 2 GB → silently fails at lower ceilings. **Bug:** `sync_orch.rs:907` passes hardcoded `MAX_FILE_BYTES` not `cfg.max_file_size_bytes`. Fix: thread config value; add `clamp_values` upper bound = effective transport ceiling; UI max reflects it. (core+daemon+ui)
- **B4 (P1) text-default comment drift** `SettingsView.tsx:351-352` comment says 15 MiB, value+core = 10 MiB. Fix comment / hydrate from `get_config`. (ui, trivial)
- **B5 (P1) three AppConfig keys have no UI/IPC setter:** `collect_public_ip`, `paste_as_plain_text`, `excluded_app_bundle_ids`. Fix: add to `IpcConfig`+`merge_config`+SettingsView (or doc as file-only). (daemon+ui)
- **B6 (P1) Android missing `max_file_size_bytes` + `sensitive_ttl_secs`** (parity). (part of B1)
- **B7 (P1) Android no floor-clamp** on quota/size → can recur the 200-byte-quota self-clearing bug macOS guards. (part of B1)
- B8–B10 (P2) `poll_interval_ms` no UI; dead/no-op consts (`sqlite_cache_mb` hardcoded, `INLINE_THRESHOLD_BYTES` unused); `image_quality` no-op for PNG on both.

## Cluster C — Cloud sync fidelity
- **C1 (P1/HIGH) cloud file sync loses file_name + mime.** `clipboard_item_to_json` (cloud.rs:3030) omits them; `build_local_blob_item` (:2826) hardcodes `file`/`application/octet-stream`. P2P preserves them → asymmetry. Fix: embed name+mime in the encrypted plaintext envelope (no schema change). (daemon)
- **C2 (MEDIUM) cloud backlog never re-sweeps after `set_sync_passphrase`.** Start daemon → enter passphrase later → pre-existing history never uploads until re-copied. Fix: trigger re-sweep on key None→Some. (daemon)

## Cluster D — Relay SSE (#26)
- **D1 (P1/HIGH) parked producer task leaks on idle-inbox disconnect.** Disconnect detected only on `tx.send` (a drain); a parked `rx.recv()` with empty inbox never wakes until next push → task+receiver held. Fix: `tokio::select!{ rx.recv(), _ = tx.closed() => return }`. + disconnect-teardown test. (relay)

## Cluster E — Android state SSOT + efficiency
- **E1 (P1) Android online-dot derived from sync cursor, not presence.** `lastSupabasePollWallTime` = wall_time of last processed row, not last-seen clock → semantically diverges from macOS (live mTLS + last_sync≤60s). Fix: stamp a real `lastSuccessfulSyncMs` on poll/connect success, or surface presence from core/FFI. (android)
- **E2 (P2) Android roster from SharedPreferences (single most-recent peer only)** → shows fewer devices than macOS at 2+ peers. Fix: back roster with the peer store the dialer uses. (android)
- **E3 (P1/HIGH efficiency) `ingestWsRow` does a full history GET per WS push** (`pollFromSupabase(0,"")`) just to get client+key, discards rows. Fix: `resolveSyncContext()` accessor returning (client,key,bearer) with no poll. (android)
- **E4 (MEDIUM) stale socket `onClosed` can null a newer `activeSocket`** → drops to 60s poll. Fix: `if (activeSocket === webSocket)` identity guard. (android)
- **E5 (MEDIUM) connect-cancel relies on unreachable `InterruptedException`** + 1s busy-wait on latch. Fix: make wait cancellable / drop dead catch. (android)
- E6 (LOW) reconnect backoff has no jitter (doc claims it) → thundering herd. E7 (LOW) cross-listener dedup keys on 32-bit hashCode.

## Cluster F — daemon/p2p robustness + buffer peaks (nits, no P0)
- **F1 (P1, latent) shutdown-token reaches only `accept_loop`;** responder/outbound/connector/discovery loops unsignaled. Benign now (start_p2p once + process exits) but a leak the moment in-process P2P restart lands. Fix: clone the existing `CancellationToken` into all five. (daemon)
- **F2 (P1) file/image rewrap buffer peak ~3-4× under DB mutex.** `sync_orch.rs:838` clones the blob (`take()` instead); move CPU-heavy re-encode out from under `db.blocking_lock()`. (daemon/core) — bounded by ceilings so not P0.
- **F3 (P2) `chunks_to_blob` realloc spike** → pre-size the output Vec. (core)
- F4 (LOW) `derive_xchacha_key` returns bare `[u8;32]` not `Zeroizing`. F5 (LOW) `BOOTSTRAP_ACCEPT_TIMEOUT` TODO drift vs QR_TTL — extract shared const.
