# v0.5.3 Bug & Improvement Backlog

> Captured 2026-05-31 from user hardware testing of v0.5.2. All items below MUST be
> fixed in v0.5.3. Per the **platform-parity** rule, every macOS fix must be mirrored
> on Android and vice versa unless the item is explicitly platform-scoped.

Status legend: `[ ]` open · `[~]` in progress · `[x]` done

---

## macOS

- [x] **M1 — Image clips render differently from text clips.**
  Image rows must use the same row layout as text rows: action buttons and the
  timestamp belong on the **right side**, consistent with text clips.

- [x] **M2 — Pin icon.** Replace the macOS pin icon with the Android pin icon (parity).

- [x] **M3 — Pinned-item background.** Change the background treatment of a pinned
  element (distinct visual for pinned state).

- [x] **M4 — Preview-lines setting is shared between popup and app.**
  The "preview lines" value is currently coupled across the popup and the main app
  window. They must be **two independent settings**, not linked.

- [x] **M5 — Remove "items displayed" setting.** Replace with lazy loading
  (infinite/virtualized list); no fixed display cap exposed to the user.

- [x] **M6 — "Preview hover delay" is broken/unclear.** If it is meant to show a
  preview on hover, it does not work. Remove the hover-preview mechanism entirely and
  replace it with an explicit **preview button**.

- [x] **M7 — Remove "Set" button in Sync Passphrase.** It appears redundant and causes
  the input fields to be misaligned. Remove it and fix field alignment.

- [x] **M8 — Hide settings descriptions behind an info icon.** Each setting's
  description should be collapsed behind an icon. Click icon → popover with the
  description; click outside the popover → close it (tooltip-like behavior).

- [x] **M9 — Inconsistent / broken sliders.** Display and Storage sections use
  different sliders; the Storage slider is broken. Replace **all** sliders with the
  Display-section slider component.

- [x] **M10 — Popup is janky.** The popup is jerky and the scroll jumps when merely
  moving the mouse. Also: popup is missing pin buttons and any visual indication that
  an item is pinned. Fix scroll/jank and add pin affordance + pinned visual.

- [x] **M11 — Enlarge popup by 1.2×.**

- [x] **M12 — Sound & notification on every copy.** Sound and notification must fire on
  **any** copy, not only copies made through the popup. Currently they only fire when
  copying an image. (Related: [[project_macos_sound_notify_rootcause]].)

## All platforms (macOS + Android)

- [ ] **A1 — Drag-to-reorder pinned items.** Allow reordering pinned items relative to
  each other via a dedicated **drag handle** element.

- [x] **A2 — "Clear all" must not delete pinned items, and must be removed from the
  main screen.** Currently it wipes everything including pinned items. (a) Pinned items
  must be preserved by clear-all; (b) remove the Clear-All button from the main screen
  entirely (move to settings/menu if kept at all).

- [x] **A3 — Copying from the main list must not scroll to top.** When copying a clip
  in the main window, the list jumps/scrolls to the top and "jitters." Keep scroll
  position stable on copy.

- [x] **A4 — Paired-device info is minimal.** Remove the Copy button next to the
  fingerprint — the fingerprint now **copies on click** on both macOS and Android.
  (last activity/sync/ping not available in current IPC — deferred.)

- [x] **A5 — Pairing QR / "already paired" text & flow.** (see screenshot)
  - Remove the "1 device is already paired. Scanning this code … will have no effect."
    text — the QR should simply regenerate when it becomes invalid.
  - Remove the **Regenerate code** button.
  - **Blur** the QR code by default; **unblur on click**; a **second click generates a
    new code**.

- [x] **A6 — Log viewer text must wrap.** In the log viewer, long lines must wrap so
  everything is visible on screen (no horizontal overflow / clipped text). Applies to
  both platforms.

- [x] **A7 — macOS log export & viewer.** macOS now has an in-app **log viewer** (Logs
  nav item) that reads `~/Library/Logs/CopyPaste/daemon*.log` and supports **Export**
  to download the log file. (Related: [[project_android_logging]].)

## Android

- [x] **AND1 — List flicker on copy.** When copying an item from the list, the whole
  list disappears and then reappears. Fixed: LoadingBox now only shows on first load
  (when list is empty), not on background refreshes.

- [x] **AND2 — Remove "Start pairing" button.** No longer needed: the QR block is now
  blur-by-default and unblurs on click (see [[A5]]). Remove the Start-pairing button and
  rely on the QR tap behavior.

- [x] **AND3 — Split settings into separate tabs.** Settings now split into
  General / Display / Storage / Sync / Notifications tabs via ScrollableTabRow.

- [x] **AND4 — Add a Save button to settings.** Settings now persist only after
  the user clicks Save (no auto-apply on change).

- [x] **AND5 — "Image max height" should be a slider, not an input field.**

- [x] **AND6 — "Auto close delay" should be a slider, not an input field.** Also clarify
  what this setting does (label/description unclear).

- [x] **AND7 — Images don't sync from macOS.** Root cause: daemon SQL backlog query
  filtered `WHERE content_type = 'text'`; Android `decryptRow` also rejected non-text rows.
  Fixed: removed text-only guards in `cloud.rs` (daemon) + `SupabaseClient.kt` / `FgsSyncLoop.kt` (Android).

- [x] **AND8 — Accessibility still not working.** Defensive fix: wrapped `onServiceConnected`
  in try/catch (previously any constructor exception silently killed the service); added
  AppLogger lifecycle entries so bind/unbind events appear in the in-app log viewer.

- [x] **AND9 — Header title must match nav-bar title.** The title shown in the header
  must correspond to the current nav-bar (bottom-nav) title.

- [x] **AND10 — Device list detail parity.** Fingerprint copies on click (same as macOS).
  Last-activity/last-sync/ping deferred — not available in current IPC response.
