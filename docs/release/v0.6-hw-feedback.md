## CRITICAL — sync

- [~] **B2 — Sync very slow (30s–1min+ latency).** A clip copied on device A takes 30s to
  over a minute to reach device B. Must be near-realtime. *Status: opus investigator tracing
  every poll/sleep on the end-to-end path (capture cadence, push trigger, relay poll interval,
  P2P push-vs-pull). Prime suspect: a fixed 30–60s relay/sync poll. → fix plan incoming.* **P0**
- [~] **B3 — Images may not sync macOS→Android at all.** User can't confirm images arrive on
  Android (possibly masked by B2 slowness). Suspected encrypt/format asymmetry: desktop stores
  XChaCha20 chunked-encrypted blobs; Android reportedly handles images as plaintext-in-prefs.
  *Status: opus investigator checking the Android receive/decrypt/store/render path.* **P0**

## macOS

- [~] **M2 — Window not draggable by header (RECURRING).** *ROOT-CAUSED:* commit `2321095` set
  `decorations:false`, which makes `titleBarStyle:"Overlay"` a no-op so the native drag/hit-test
  is gone (the `data-tauri-drag-region` strips at App.tsx:184 / Sidebar.tsx:155 are already correct).
  *Fix:* revert `tauri.conf.json:24` → `"decorations": true` (keep Overlay + hiddenTitle); also
  restores traffic-light buttons. JS `startDragging()` fallback if needed. → TODO implement. **P1**
- [~] **M4 — Private-mode tray shows wrong state (RECURRING).** *ROOT-CAUSED:* the tray
  `CheckMenuItem` is read once at startup and never refreshed when private mode is toggled from the
  Settings window (polarity/field are correct — it's a stale cache). *Fix:* emit a
  `private-mode-changed` event from SettingsView.tsx:810 (with the daemon-confirmed value) + a Rust
  `app.listen` in lib.rs (~235) that calls `set_checked` on the stored `PrivateModeMenuItem`; have
  Settings also listen so tray+settings+daemon converge. → TODO implement. **P1**
- [~] **M1 — Lazy-load modal/popup to cut memory.** The static "popup" WKWebView is created at
  launch and never destroyed (~84MB of the ~350MB idle). *Status: opus investigator designing a
  free-on-hide / lazy strategy that keeps quick-paste show-latency low.* **P1**
