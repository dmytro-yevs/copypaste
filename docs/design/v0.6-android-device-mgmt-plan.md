## Findings (why it's broken)
- Android stores exactly ONE peer in SharedPreferences (`Settings.kt` pairedPeerFingerprint:488 / pairedPeerSyncAddr:499 / pairedPeerSessionKey:515, KEK-wrapped). `PairActivity.runPairAndSync:397-400` OVERWRITES on each pair → pairing a 2nd Mac discards the 1st.
- No persistent mTLS allowlist on Android: trust is rebuilt per call inside `sync_with_peer` (`crates/copypaste-android/src/lib.rs:753-755` — `PairedPeers::new(); add(peer_fingerprint)`). The single passed fingerprint IS the whole allowlist. No inbound listener exists yet.
- Core revoke primitives EXIST but are UNREACHABLE from Android: `copypaste_core::{ensure_revoked_devices_table, revoke_device, revoke_devices, list_revoked_devices}` (`storage/devices.rs:39/64/108/151`), re-exported (`core/src/lib.rs:45`). UDL exposes none. Android opens SQLCipher `copypaste.db` (`ClipboardService.kt:709`, gated by `android-uniffi-live` feature) — correct home for the `revoked_devices` table.
- **BUG (current):** `DevicesActivity.revokePeer` (`:121-141`) only mutates Settings AND sets `p2pIdentity = null` — wiping OUR OWN identity (new cert/fingerprint), which breaks every OTHER paired peer that pinned us. Revoke must target the PEER, not self.

## Step A — Rust/FFI (ONE android-crate change; bump `version.rs` UNIFFI_ABI_VERSION 8→9; regen ONCE)
Files: `crates/copypaste-android/{uniffi/copypaste_android.udl, src/lib.rs, src/version.rs}`. Fold ALL ABI breaks into this one commit (also the queued `device_id` param fixwave at lib.rs:669-677), so bindings regen happens once.

1. **`revoke_device_audit(db_path, key, fingerprint, name) -> u64`** — `with_cached_db` → `copypaste_core::revoke_device(conn, &fp, &name)` (writes `revoked_devices` row, returns revoked_at). Feature-gate like the live-DB calls; `#[cfg(not(android-uniffi-live))]` stub returns a timestamp without I/O.
2. **`list_revoked_fingerprints(db_path, key) -> sequence<string>`** (or richer `dictionary RevokedPeer { string fingerprint; string name; i64 revoked_at; }` for an audit list — recommended for parity) → `list_revoked_devices(conn)`.
3. **`sync_with_peer` gains `revoked_fingerprints: sequence<string>` (and `device_id: string`).** At the TOP of the fn, before building `PairedPeers` (lib.rs:753), canonicalize (lowercase, strip ':') and if `peer_fingerprint` is in the denylist → `Err(P2pError "peer … is revoked")`. THIS is the security-critical, load-bearing control (Android analog of daemon `PairedPeers::remove`).

## Step B — regen once
`./scripts/generate-android-bindings.sh`; verify `copypaste_android.kt` has `revokeDeviceAudit`, `listRevokedFingerprints`, new `syncWithPeer` sig; crate tests (`uniffi_binding_fields.rs`, conformance) pass. Bump the Kotlin compiled-in ABI const to 9 (checkCompatibility, copypaste_android.kt:1865) — Kotlin won't compile until then.

## Step C — Kotlin (after B; partition by FILE to avoid agent conflicts)
Agreed contract so agents don't collide:
- **`Settings.kt` (agent owns this file):** add `data class PairedPeer(fingerprint, syncAddr, name, sessionKeyWrappedB64, sessionKeyIvB64, lastSyncMs=0)`; roster backed by JSON pref `paired_peers_json` (Android mirror of peers.json). API: `var pairedPeers: List<PairedPeer>`, `upsertPeer(p)` (append/replace by fp — the overwrite fix), `removePeer(fp)`, `sessionKeyFor(fp): ByteArray` (KEK-unwrap, reuse wrapKey/unwrapKey:778/785). MIGRATION: if `paired_peers_json` absent but legacy `paired_peer_fingerprint` non-blank → synthesize one PairedPeer from the 3 legacy fields, write JSON. Keep legacy scalar getters as shims over `pairedPeers.firstOrNull()` OR migrate all callers.
- **`CopypasteBindings.kt`:** Kotlin wrappers for `revokeDeviceAudit`, `listRevokedFingerprints`, new `syncWithPeer` sig (toUByteList/toByteArray helpers, try/catch→CopypasteException). Bump Kotlin ABI const to 9.
- **`PairActivity.kt`:** `runPairAndSync:397-400` → `settings.upsertPeer(PairedPeer(...))` (append). Paired-device card (658-695) reads from `settings.pairedPeers`.
- **`FgsSyncLoop.kt`:** `dialPairedPeer` loops over `settings.pairedPeers`; load `val revoked = listRevokedFingerprints(...)` once; skip revoked (local fast-skip) AND pass `revokedFingerprints=revoked` into `syncWithPeer` (transport refusal); per-peer try/catch so one failure doesn't abort others.
- **`DevicesActivity.kt`:** multi-peer list, per-peer Unpair + Revoke (macOS DevicesView parity). `unpairPeer(fp)=settings.removePeer(fp)`. `revokePeer(ctx,settings,peer){ settings.removePeer(fp); revokeDeviceAudit(dbPath, key, fp, name) on Dispatchers.IO }`. **DELETE the `p2pIdentity = null` line (the bug).** Update confirm dialog copy.

## Security (must hold)
1. Revocation enforced at the TRUST layer (Step A3 denylist in sync_with_peer), not just UI. 2. Any future Android inbound mTLS listener MUST consult the same `revoked_devices` denylist in its accept path (mirror transport.rs:431). 3. Revoke must NOT wipe own identity. 4. Per-peer session keys stay KEK-wrapped in the roster JSON. 5. `revoked_devices` lives in SQLCipher `copypaste.db` under `settings.encryptionKey`; handle non-live build (stub) gracefully.

## Order
Rust A (one commit, sole android writer) → regen B → Kotlin C (Settings first → bindings wrappers → PairActivity/FgsSyncLoop/DevicesActivity). Sequence the Android FFI AFTER the daemon/core waves (W2/W3) merge, and AFTER (or combined with) the AppConfig-over-FFI bridge (W6) since both bump ABI — ideally ONE FFI agent adds config + revoke + sync_with_peer params together → one ABI bump → one regen.
