# v0.6 â€” Image & File Sync (authoritative plan)

> 2026-06-03. Images/files are UNIMPLEMENTED in the sync layer (text-only). This plan makes
> image AND file sync work both directions, P2P + cloud. Base: `v0.6-integration` @ d4af7cd.

## Root cause (verified)
- Storage works: `handle_image` (daemon.rs:1518) stores XChaCha20 512KB chunks under the **device
  local key (v1 seed)** + content-hash `file_id`; `content_type="image"`, `content`=chunk blob,
  `blob_ref`=meta JSON (file_id/dims/chunk_count). Files: `core::file.rs` `encode_file` (same shape).
- **Send broken:** `sync_orch.rs:591` `rekey_outbound` returns `NotApplicable` for non-text â†’
  images/files never in catch-up set (`:539`); live fanout forwards the at-rest blob still wrapped
  in the **sender's local key** â†’ peer can't decrypt (verifier blocker 8a).
- **Receive broken:** Android `lib.rs:827` drops every non-text frame; no decrypt path.
- **Cloud skips:** `cloud.rs:846` marks non-text rows synced-and-skipped; `cloud.rs:1239/2646` reject.
- `WireItem` (protocol.rs:26) has no explicit file_id (only inside blob_ref).

## Decision: Option 2 â€” re-wrap plaintext under the shared key (reuse the text re-key path)
On the wire, an image/file travels as a SINGLE blob: `content = encrypt_for_cloud(shared_key,
item_id, plaintext_bytes)`, `content_nonce = None`, `content_type` preserved, `blob_ref = None`.
No per-chunk re-key, no wire file_id, no new Android chunk scheme. item_id is deterministic
(uuid-of-file_id) so AAD matches on both ends and LWW/dedup converge.

## THE WIRE CONTRACT (both daemon and Android must follow exactly)
**rekey_outbound (send), content_type image|file:**
1. parse file_id from `blob_ref` meta JSON.
2. plaintext = imageâ†’`chunks_from_blob(content)`+`decode_image(chunks, v1_key, file_id)`;
   fileâ†’`chunks_from_blob`+`decode_file(chunks, v1_key, file_id)`. (chunks were encrypted under the
   local **v1 seed**, NOT derive_v2 â€” see daemon.rs:1521.)
3. if `plaintext.len() > SYNC_MAX_BLOB_BYTES` (recommend 8 MiB) â†’ return `Failed` (skip, log warn) to
   respect the 16 MiB P2P frame (transport.rs:79) and cloud body limits.
4. `wire.content = encrypt_for_cloud(shared, wire.item_id, &plaintext)`; `content_nonce=None`;
   `blob_ref=None`; keep `content_type`. Return `Rewrapped` (â†’ catch-up :539 + fanout :164 include it).

**rekey_inbound / Android inbound (receive), content_type image|file, content present + nonce None:**
1. `plaintext = decrypt_from_cloud(shared, item_id, content)`.
2. macOS: re-chunk under local key â€” imageâ†’`encode_image_with_limit(plaintext, v1_key,
   file_id=image_content_hash(plaintext), max, max_decoded_mb)`; fileâ†’`encode_file(...)` â†’
   `chunks_to_blob` + rebuild meta JSON (as handle_image daemon.rs:1566) â†’ store. content_type kept.
3. Android: emit `SyncedItem{content_type, plaintext: bytes, item_id, blob_ref?}`; Kotlin stores
   (imageâ†’`storeImageBytes`, fileâ†’file store) + renders.
