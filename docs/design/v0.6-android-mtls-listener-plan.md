# v0.6 — Android inbound mTLS P2P listener (so macOS can INITIATE to Android)

> 2026-06-03 (investigator ac2be7e2, read-only). Today Android only dials out + polls; no inbound accept.
> This adds a persistent inbound mTLS listener at parity with the macOS daemon's accept_loop. ABI 10→11.

## Reference (macOS): `copypaste-daemon/src/p2p.rs::accept_loop` (509-609)
Needs: bind `0.0.0.0:port` (port 0 = OS-assigned, read `local_addr().port()`); device mTLS identity (cert/key);
`PairedPeers` allowlist (TLS verifier `is_known`, transport.rs:431 rejects unpinned); per-conn pump
`run_peer_connection_framed` reading length-delimited JSON `WireItem` frames; push catch-up on accept (p2p.rs:574);
revoked eviction via `PairedPeers::remove`. Wire protocol = bare length-delimited JSON WireItem (NOT SyncEngine::run_session).

## Android crate already links copypaste-p2p/sync/core + tokio + tokio-util(codec). ADD `"sync"` to tokio-util features (Cargo.toml:56).

## FFI shape — blocking cancellable listener via a process-global registry (UDL has no interface objects)
New module `crates/copypaste-android/src/p2p_listener.rs`:
- statics `LISTENER_REGISTRY: OnceLock<Mutex<HashMap<u64, ListenerHandle>>>`, `NEXT_LISTENER_ID: AtomicU64` (same idiom as RUNTIME/DB_HANDLES).
- `ListenerHandle { cancel: CancellationToken, actual_port: u16, received: Arc<Mutex<Vec<SyncedItem>>> }`.
New FFI funcs (lib.rs + UDL):
1. `start_p2p_listener(listen_port:u16, cert_der, key_der, allowed_fingerprints:[String], revoked_fingerprints:[String], session_keys:[PeerSessionKey], local_items:[LocalItem], device_id:String) -> P2pListenerHandle{listener_id:u64, actual_port:u16}` — binds, registers, spawns accept loop on `runtime()`, returns immediately so Kotlin learns the port.
2. `poll_p2p_listener(listener_id:u64) -> [SyncedItem]` — atomically drains `received`. Kotlin stores via the SAME `storeItemWithLww`/`storeImageBytes`/`storeFileBytes` paths as the dialer (reuse SyncedItem → zero new store code; LWW dedups the dial/listen overlap).
3. `update_p2p_listener_peers(listener_id, allowed, revoked, session_keys)` — live roster/denylist update (PairedPeers add/remove) without restart.
4. `stop_p2p_listener(listener_id:u64)` — cancel() + deregister; idempotent.
New dicts: `PeerSessionKey{fingerprint:String, session_key:[u8]}`, `P2pListenerHandle`.

## Accept loop (port of accept_loop, simplified — no outbound fanout)
`select! { accept => {...}, cancel.cancelled() => break }`. Per accept: **re-check `is_fingerprint_revoked(peer_fp, revoked)` BEFORE catch-up/any frame** (mandatory — else a revoked peer gets the history push); derive shared key via `shared_sync_key_from_session` keyed by the VERIFIED peer_fp; spawn: (1) push catch-up (re-key local_items → WireItem, same as sync_with_peer lib.rs:1079-1140), (2) persistent read loop (NO idle/deadline cutoff — keep link open like daemon), decrypt_from_cloud per frame → push SyncedItem into `received`.

## SECURITY (load-bearing, mirror macOS)
- Allowlist pinning IS the authenticator: seed PairedPeers ONLY from `settings.pairedPeers`; unpinned fp rejected at TLS (transport.rs:431).
- Denylist enforced at ACCEPT, not just dial (inbound analog of lib.rs:1047). MANDATORY.
- Per-peer session key (frame from A decrypted with A's key) → `Vec<PeerSessionKey>`, not a global key.
- Bind 0.0.0.0 (LAN); INTERNET perm already present; no new perm.

## Discoverability — makes the listener reachable
- **Path A (v1, minimal, NO NSD):** at pair time Android passes its `"<lan-ip>:<actual_port>"` as `sync_addr` to the macOS responder (`bootstrap_pair_initiator` sync_addr param), which macOS persists into peers.json → its connector dials back. Caveat: Android DHCP IP change needs re-pair/manual refresh (Android isn't advertising).
- **Path B (follow-up):** Android `NsdManager.registerService` of `_copypaste._tcp` advertising actual_port + device_id TXT → macOS DiscoveryService auto-refreshes the address. Kotlin-only, independent of the FFI. DEFER to a follow-up wave.

## Kotlin / FGS (ClipboardService owns it — START_STICKY, has scope + FgsSyncLoop + realtimeClient)
- Start in onStartCommand after fgsSyncLoop.start, gated on syncEnabled && p2pSyncEnabled; cert from DeviceKeyStore.peek(); allowed=pairedPeers; revoked=listRevokedFingerprints; sessionKeys=sessionKeyFor(each); call startP2pListener, store listenerId.
- Drain: a coroutine (or fold into FgsSyncLoop sleep loop) polls `pollP2pListener(listenerId)` on the dial cadence and runs the SAME store-mapping (factor `storeSyncedItem(item)` out of dialPairedPeer FgsSyncLoop.kt:476-552).
- Stop in onDestroy alongside fgsSyncLoop.stop/realtimeClient.close/scope.cancel. Idempotent re-bind on sticky restart.
- Doze: socket lives only while FGS alive; outbound dialer + Supabase poll are the Doze fallbacks. No WakeLock. LWW dedup handles double-sync.
